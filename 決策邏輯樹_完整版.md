# 決策邏輯樹 | 完整版

## 框架總覽

```
支撐壓力決策框架
│
├─ 短期（5日）支撐/壓力
│   ├─ 最小距離：3%
│   ├─ Tier來源：5日極值、POC、MA5、布林通道
│   ├─ 降級目標：5日 → 20日 → 整數防線 → 最遠值
│   └─ 整數防線：0/5尾數，範圍±5%
│
├─ 中期（20日）支撐/壓力
│   ├─ 最小距離：7%
│   ├─ Tier來源：20日極值、POC、MA20、布林通道
│   ├─ 降級目標：20日 → 60日 → 整數防線 → 最遠值
│   └─ 整數防線：0尾數，範圍±12%
│
└─ 長期（60日）支撐/壓力
    ├─ 最小距離：10%
    ├─ Tier來源：60日極值、POC、MA60、布林通道
    ├─ 降級目標：60日 → 120日 → 240日 → 整數防線 → 最遠值
    └─ 整數防線：世紀大關，範圍±18%
```

### 🔍 七個階段的白話詳解

#### 階段1：Tier 1-5搜尋 | 「用最好的指標去找」

拿出最可靠的五套指標，按優先級逐一檢查看有沒有符合條件的支撐或壓力。

**五套指標的優先級：**
```
Tier 1（最優先）：5日最低點/最高點、VAL/VAH
Tier 2：POC
Tier 3：MA5、布林通道
```

**搜索邏輯：**
按Tier順序檢查 → 找到符合的就用Tier 1的 → 如果Tier 1都不符，再看Tier 2 → 依此類推

**成功條件（很嚴格）：**
- 邏輯必須正確：支撐必須 < 現價，壓力必須 > 現價
- 距離必須 ≥ 3%：距離現價至少要3%以上（排除短期雜訊）

---

**如果同一Tier有多個符合，怎麼選？**
在符合條件的支撐/壓力中，選離現價最近的。

原因：
1. **可達性好**：支撐離現價近，股價容易觸及，反彈效果更快被驗證
2. **不失可靠性**：都符合最小距離要求，邏輯和品質一樣好
3. **實戰優勢**：短期交易者更需要「快速有效」的支撐壓力

**如果成功：**
✓ 找到了就直接用，這叫「首選」，最優質的結果

**如果失敗：**
→ 降級到階段2，找次優選項

---

#### 階段2：Tier 1邏輯檢查 | 「放寬要求，只看邏輯」

階段1失敗了，就回頭看「最關鍵的指標」，但這次不要求距離，只要邏輯正確。即使距離不足3%，也比不符合邏輯的更可靠。

**只看這兩個：**
- **支撐**：5日最低點、VAL中「< 現價」的，取最高值
- **壓力**：5日最高點、VAH中「> 現價」的，取最低值

**如果階段2成功：**
✓ 找到了，這叫「次選」，品質還不錯

**如果失敗（創新高/創新低）：**
→ 降級到階段3

---

#### 階段3：框架極值 | 「用這個時間框架內的真實高低點」

階段1和2都失敗了？那就直接用「5天內的極值」，無論它是否在Tier 1中。
實際上，階段3往往會被跳過，因為階段2已經涵蓋了

**為什麼要保留階段3？**

防止邏輯漏洞。即使理論上階段2和3重複，但保留階段3確保：
1. 邏輯完整性：不會遺漏任何「邏輯正確」的候選
2. 清晰度：階段明確分工，不易出錯

**實際執行時：**
- 大多數情況：階段2成功，階段3被跳過
- 創新高/低情況：階段1、2都失敗，階段3也失敗，直接跳到階段4

**如果成功：**
✓ 找到了，這叫「降級」（已經開始降級了，品質下降中）

**如果失敗（創新高/低）：**
→ 降級到階段4（升級到更大時間框架）

---

#### 階段4：更大框架 | 「看20天而不是5天」

5天的數據都不夠用了？那就擴大看20天。

**支撐選法：**
取20日來的最低點（只要它 < 現價）

**壓力選法：**
取20日來的最高點（只要它 > 現價）

**如果成功：**
✓ 找到了，還是「降級」品質

**如果失敗：**
→ 短期無階段5，直接跳到階段6

---

#### 階段5：級聯框架 | 「長期獨有，短期跳過」



#### 階段6：整數防線 | 「交易者喜歡整數，所以整數位置有支撐」

前面的技術指標都失敗了？那就找「整數」位置。為什麼？因為心理學。

**短期搜索規則：**
```
現價 = P
Tier最遠值 = T（前面找到的最遠指標值）

搜索範圍：T × (1±5%)
搜索目標：在範圍內找0/5尾數

支撐：找「< T」的最大0/5尾數
壓力：找「> T」的最小0/5尾數

例子：T = 102
支撐範圍：96.9 ~ 102，尋找：100、95...
壓力範圍：102 ~ 107.1，尋找：105、110...
```

**如果成功：**
✓ 找到了，這叫「防線」，是心理層面的支撐

**如果失敗：**
→ 降級到階段7（最後保障）

---

#### 階段7：最遠值 | 「保證一定有輸出」

上面六個階段全部失敗？那就用「所有候選中最遠的那個」。

**支撐選法：**
把Tier 1-5的所有值、5日、20日的高低點 全部蒐集起來，取最小值。

**壓力選法：**
把Tier 1-5的所有值、5日、20日的高低點 全部蒐集起來，取最大值。

## 決策流程比較

### 三框架的共同流程

