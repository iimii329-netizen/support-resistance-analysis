## 一、回測邏輯說明

### 1.1 演算法架構
三層級決策樹，依序執行 Layer 1 → Layer 2 → Layer 3，取第一個有效候選。

| 層級 | 名稱 | 資料來源 | 觸發條件 |
|------|------|--------|--------|
| **L1** | Volume Profile | 5/20/60日周期 VP（POC/VAH/VAL） | 數據可用且通過距離篩選 |
| **L2** | Price Action | Fractal 波段高低點 | L1 無有效候選時 |
| **L3** | Moving Average | MA 線與整數關卡 | L2 無有效候選時（保底層） |

### 1.2 有效性判定邏輯（修正版 — 分兩階段）

**支撐線有效：**
```
第一階段（觸線日 — 掃描未來 N 天）
  IF 最低價 ≤ 支撐線 × 1.005 THEN 記錄觸線日
  
第二階段（確認日 — 從觸線日起掃描 M 天）
  IF 收盤價 ≥ 觸線日MA × (1 + threshold) THEN 有效
  
評估窗期：
  - 5日週期：觸線日後 2 天內確認
  - 20日週期：觸線日後 5 天內確認
  - 60日週期：觸線日後 10 天內確認
```

**壓力線有效：**
```
第一階段（觸線日 — 掃描未來 N 天）
  IF 最高價 ≥ 壓力線 × 0.995 THEN 記錄觸線日
  
第二階段（確認日 — 從觸線日起掃描 M 天）
  IF 收盤價 ≤ 觸線日MA × (1 - threshold) THEN 有效
  
評估窗期：
  - 5日週期：觸線日後 2 天內確認，跌破 MA × 0.985
  - 20日週期：觸線日後 5 天內確認，跌破 MA × 0.975
  - 60日週期：觸線日後 10 天內確認，跌破 MA × 0.965
```

**市況判斷邏輯：**
- 若 ADX < 25 → RANGE（盤整）→ L1 選 VAH/VAL
- 若 ADX ≥ 25 且 close > MA5 > MA20 → TREND_UP → L1 選 VAH（當前）和 VAH（前期）
- 若 ADX ≥ 25 且 close < MA5 < MA20 → TREND_DOWN → L1 選 VAL（當前）和 VAL（前期）

**距離門檻篩選：**
```
distance = abs(candidate_price - current_price) / current_price

5日週期：distance ≤ 5% → 接受 | > 5% → 跳過
20日週期：distance ≤ 10% → 接受 | > 10% → 跳過
60日週期：distance ≤ 18% → 接受 | > 18% → 跳過
```

### 4.3 市況分組結果（路徑A 核心發現）

| **60日** | 53.3% | 4.4% ❌ | 0.0% | **66.7%** ✅ | 0.0% | 6.1% |

### 4.4 結果分析

**三個重大發現：**

**發現一：演算法選線方向正確時，有效率已接近目標**
- 多頭 + 5日支撐：77.8%、多頭 + 20日支撐：73.3% → 已達 70% 目標
- 空頭 + 20/60日壓力：66.7% → 接近目標
- **說明演算法選線邏輯本身可信，問題在其他環節**

**發現二：「逆勢方向」的線條結構性無效**
- 多頭市場中壓力線有效率 < 5%（觸線後繼續漲，不回落）
- 空頭市場中支撐線有效率 < 17%（觸線後繼續跌，不反彈）
- 這是市場結構造成的，不是選線錯誤

**發現三：盤整市場是最大問題（佔樣本 66%，有效率僅 6.4%）**
- 盤整中支撐和壓力有效率都低（9.1% / 3.7%）
- 盤整佔樣本最多，拉低整體有效率最多
- 原因待查：可能是選線邏輯（VAH/VAL）在盤整中失效，或有效性定義不適合

## 五、問題根本原因

### 5.1 壓力線整體有效率極低（5.6%）

**根本原因：** 非單一問題，是市況加權的結果

```
壓力線整體 5.6%  =
  多頭市場（270筆）壓力  2.2%  × 權重 50%
  空頭市場（36筆） 壓力  61.1% × 權重  6.7%
  盤整市場（594筆）壓力  3.7%  × 權重 66% (最重)
```

- 盤整樣本佔比最高，且有效率最低，導致整體被壓低
- 多頭市場下壓力線結構性無效（不是演算法問題，是定義問題）
- **結論：** 不應該在多頭市場對壓力線套用「觸線後回落」的定義

### 5.2 盤整市場有效率全面低落（6.4%）

**原因：** 兩個可能方向待驗證

1. **選線邏輯問題：** 盤整時 L1 選 VAH/VAL，但若 VAH/VAL 與現價差距大，
   大量線條通過距離篩選但實際上是「虛假位階」
2. **有效性定義問題：** 盤整市場中 MA 漲跌幅本身就小，
   「收盤站上 MA×1.015」這個門檻在盤整時可能永遠達不到

### 5.3 60日週期無改進（12.0%）

**原因：** 距離門檻（18%）太寬鬆 + 盤整期 60日完全失效（0.0%）
- 盤整下 60日支撐：0/99 = 0.0%，等於 60日支撐線在盤整中完全無效

## 六、修正內容說明

### 6.1 Bug 修正 — 日期對齊

**問題：** 兩個 DataFrame（ohlcv_df 和 indicator_df）的起始日期不同，用位置索引對齊導致取錯資料

```
原結構：
  ohlcv_df[0]      indicator_df[0]
  2025-01-02    vs 2024-05-08      ← 日期不符，導致後續都錯

修正後：
  用日期作為 index，查詢時直接用日期 key
  避免位置索引錯配問題
```

**影響範圍：** MA5/20/60 基準值、ADX 判斷、市況分類

### 6.2 邏輯修正 — 觸線與確認分離

**問題：** 原邏輯要求「同一根 K 棒最低觸線 + 同日收盤就站上 MA」幾乎不可能

```python
# 修正前（邏輯衝突）
if low <= line.price and close >= ma * 1.015:
  return valid  # 要求先跌後漲在同一根 K 棒成立

# 修正後（分兩階段）
# 第一階段：掃描 N 天內有無觸線日
if low <= line.price:
  touch_idx = idx  # 記錄觸線日
  
# 第二階段：從觸線日起，掃描 M 天內有無確認日
if close >= ma * 1.015:
  return valid  # 允許觸線和確認在不同 K 棒
```

**效果：** 支撐線有效率 22% → 26.7%，觸線條件更容易觸發

## 七、下一步計畫

### 7.1 核心策略轉向 — 有效性定義納入市況

**原始思路（已推翻）：**
用同一套有效性定義跑所有市況 → 整體 16.1%

**新思路（路徑A 結論）：**
有效性的判定必須考量市況，因為支撐與壓力的「有效行為」在不同市況中是相反的

```
多頭市場：支撐有效 → 正常（觸線後反彈）
          壓力有效 → 幾乎不可能（觸線後繼續漲）

空頭市場：壓力有效 → 正常（觸線後回落）
          支撐有效 → 幾乎不可能（觸線後繼續跌）

盤整市場：支撐/壓力都有效，但目前 MA 確認門檻不適合盤整環境
```

### 7.2 方向三 — 解決盤整市場失效（最高優先）

**問題：** 盤整佔樣本 66%，有效率僅 6.4%，影響整體最大

**根因假設：**
- 盤整中 MA 波動幅度小，收盤站上 MA×1.015 的門檻太高
- 改用「觸線後反轉幅度」代替「站上 MA」，適合盤整環境

**新定義：**
```
盤整市況下：
  支撐有效 = 觸線後 N 天內收盤回彈 ≥ 觸線日 close × threshold
  壓力有效 = 觸線後 N 天內收盤回落 ≥ 觸線日 close × threshold
```

### 7.3 方向四 — 多空市況下的「對向線條」處理

**問題：** 多頭市場的壓力線、空頭市場的支撐線，現行定義下幾乎不可能有效

**選項一（保守）：** 在輸出時標示市況，由用戶判斷
**選項二（激進）：** 在多頭市場中停止計算壓力線有效率，改計算「突破確認有效率」
