### 1.2 存在的關鍵問題（必須解決）

#### 問題 0：L1 選層未含區位判定與鄰近失效機制（優先級 🔴 最高）

**症狀：**
```
v2.1 L1 選層存在兩項缺陷：

缺陷 1（方向誤判）：
  選層直接由市況判定 L1 的 VP 來源
    TREND_UP  → 支撐選 VAH（不論 Close 位置）
    TREND_DOWN→ 壓力選 VAL（不論 Close 位置）
  產生四種誤判：
    A：TREND_UP + Close < VAH → VAH 實為壓力，誤當支撐
    B：TREND_DOWN + Close > VAL → VAL 實為支撐，誤當壓力
    C：RANGE + Close < VAL → VAL 已失效（跌破），仍當支撐
    D：RANGE + Close > VAH → VAH 已失效（突破），仍當壓力

缺陷 2（距離無效）：
  Close 貼近 L1 線時仍輸出信號，例如：
    Close = 15,498、VAH = 15,500 → 距離 0.013%
  操作空間近乎為零，且信號在 Zone 邊界來回震盪（whipsaw）

影響規模：估計 15~25% 的 L1 信號存在方向誤判或距離無效
```

**成因：**
v2.0/v2.1 設計時只考慮「市況」維度，未考慮：
1. 股價與 VP 的「相對位置」（造成缺陷 1）
2. 股價與 L1 線的「相對距離」（造成缺陷 2）

Close 可能在 VAH 之上、POC 附近、VAL 之下等四種區位，每種區位對應不同的 VP 角色；此外，即便方向判斷正確，貼得太近的 L1 線也無操作價值。

**必要行動：**
- 在市況判定後、選層前插入「價格區位判定」階段，並於 L1 輸出前追加「VP 線鄰近失效」檢查 → Phase 1-0（新增最高優先任務，兩項同批實作）
- 架構書已升版為 v2.2（區位感知 + 鄰近失效）
- 回測必須重跑 → Phase 2-2 連帶執行
- 本任務阻擋 Phase 1-1、1-2、1-3（因為選層邏輯已變更）

#### 問題 2：盤整期有效率偏低（優先級 🟡 中）

**症狀：**
```

```

```
結果：

→ 許多假突破被誤判為有效
```

**必要行動：**
- 動態調整確認門檻（盤整 3~5%，多頭 1.5%） → Phase 2-3
- 或改用 Bollinger Bands 替代 MA 確認 → Phase 1-3（備案）

### 3.1 Phase 1：問題診斷與方案設計（10個工作日）

#### 任務 1-0：價格區位判定 + VP 線鄰近失效機制實作（2026-04-25，最高優先）

**目標：** 同批實作兩項機制，消除 v2.1 的方向誤判（4 種）與距離無效（貼近 VP 線）

**執行內容：**

1️⃣ **實作區位分類函式**
```python
def classify_price_zone(close, vah, poc, val):
    if vah is None or poc is None or val is None:
        return "ZONE_UNKNOWN"
    if close > vah:    return "ZONE_1"   # 突破價值區上緣
    elif close > poc:  return "ZONE_2"   # 價值區上半
    elif close > val:  return "ZONE_3"   # 價值區下半
    else:              return "ZONE_4"   # 跌破價值區下緣
```

2️⃣ **實作鄰近失效函式（v2.2 新增）**
```python
def is_line_too_close(close, line_price, atr, k=0.3):
    if line_price is None or atr is None or atr == 0:
        return False
    return abs(close - line_price) < k * atr
```

3️⃣ **暴露 ATR(14) 為可查詢欄位**
- 修改 `DATA/calc_adx.py`、`calc_adx_2317.py`、`calc_adx_multi.py`
- 現有 ATR 為 ADX 計算中間量，需將其追加輸出為獨立 `atr_series` 欄位
- 選層時透過 `get_atr(date)` 取得

4️⃣ **修改選層函式**
- `_select_support()` / `_select_resistance()` 加入 zone、close、atr 參數
- L1 的 VP 來源改由區位 map 決定：
  ```
  Zone 1：支撐=VAH，壓力=無
  Zone 2：支撐=POC，壓力=VAH
  Zone 3：支撐=VAL，壓力=POC
  Zone 4：支撐=無，壓力=VAL
  ```
- L1 命中後追加 `is_line_too_close` 檢查，觸發時標記 `proximity_filtered=True` 並降 L2

5️⃣ **更新輸出 JSON 欄位**
- 新增：`price_zone`、`close_at_generation`、`vah_at_generation`、`poc_at_generation`、`val_at_generation`、`atr_at_generation`、`vp_proximity_filtered`

6️⃣ **單元測試覆蓋（共 30 組）**
- Zone 判定：4 種 Zone × 3 種市況 = 12 組
- 鄰近失效：4 種 Zone × 2（近/遠）× 支撐/壓力 = 16 組
- ATR 缺失容錯：ATR=None、ATR=0 共 2 組
- 4 種誤判情境反測（A/B/C/D 各驗證修正後 L1 選擇正確）
- VP 缺失容錯測試（ZONE_UNKNOWN 降級 L2）

**交付物：**

- `test_price_zone_proximity.py`（單元測試套件，合併 30 組）

- `誤判與鄰近失效反測驗證報告.md`（4 種誤判 + 鄰近失效情境驗證）

```
現象：
  盤整有效率：18.3%（多頭 47% 的 39%）
  
假說檢驗：
  A. MA 確認門檻太寬 (1.5~3.5%)

  B. 觸及判定容差太寬 (5~18%)
     → 導致邊界值被錯判為觸及
     
  C. 支撐/壓力本身選層不當
     → L1/L2 在盤整期選擇不當
     
  D. 盤整本質無法用支撐壓力判斷
     → 盤整期雙向震蕩，邏輯註定失效

實驗設計：
  ├─ 改變容差：5→8%，觀察觸及率
  ├─ 改變門檻：1.5%→3%, 3%→5%，觀察確認率
  ├─ 改變優先層：L1→L2，觀察有效率
  └─ 比較三維度變化幅度 → 找出最重要因子
```

**步驟2：改進方案設計**

**方案 A：動態門檻法（推薦）**
```
現狀：
  多頭 MA 門檻 1.5%（好）
  盤整 MA 門檻 1.5%（太寬，易誤判）

改進：
  TREND_UP / TREND_DOWN  → 1.5%（維持）
  RANGE                  → 3.0%~5.0%（提高）
  
效果預期：
  - 盤整有效率 18% → 20~25%（提升 2~7%）
  - 但可能過濾掉小幅真實突破
  
複雜度：★☆☆☆☆ 低（只改一個參數）
實現時間：1 天
```

**方案 B：Bollinger Bands 替代法**
```
現狀：
  使用 MA ± 1.5% 為確認門檻

改進：
  使用 Bollinger Bands
  ├─ Upper Band = MA + (2 × 標準差)
  ├─ Lower Band = MA - (2 × 標準差)
  └─ 確認標準：穿越 Band 邊界

  - 自適應波動，更符合盤整特性
  - 有效率預期 18% → 22~28%（提升 4~10%）

實現時間：3 天
風險：需重新驗證
```

```

  盤整期支撐 29%、壓力 7%（不均衡）
```

```python
def validate_signal(support_price, resist_price, date, period, market_state):
    """兩階段驗證（含動態門檻）"""
    
    # 根據市況決定 MA 確認門檻
    if market_state == 'RANGE':
        ma_threshold = 0.03 ~ 0.05  # 盤整：3~5%（提高）
    else:
        ma_threshold = 0.015        # 多頭/空頭：1.5%（維持）
    
    # 第一階段：觸及判定（容差不變）
    touched = check_touch(support_price, resist_price, date, period)
    
    if not touched:
        return {'is_valid': False}
    
    # 第二階段：確認判定（動態門檻）
    ma_value = get_moving_average(date, period)
    
    # 支撐確認
    if support_touched:
        support_valid = close(touched_date) > ma_value × (1 + ma_threshold)
    
    # 壓力確認
    if resist_touched:
        resist_valid = close(touched_date) < ma_value × (1 - ma_threshold)
    
    return {
        'is_valid': support_valid or resist_valid,
        'ma_threshold_applied': ma_threshold,
        'market_state': market_state
    }
```

```python
def get_ma_threshold(market_state, volatility):
    """動態門檻計算"""
    base_threshold = {
        'TREND_UP': 0.015,
        'TREND_DOWN': 0.015,
        'RANGE': 0.030  # 盤整基礎為 3%
    }
    
    threshold = base_threshold.get(market_state, 0.015)
    
    # 盤整期根據波動度微調
    if market_state == 'RANGE':
        if volatility > 2.0:  # 強波動
            threshold = 0.050  # 提至 5%
        elif volatility > 1.0:  # 中波動
            threshold = 0.040  # 提至 4%
        else:  # 弱波動
            threshold = 0.030  # 保持 3%
    
    return threshold
```
